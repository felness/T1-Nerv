# Система распознавания текста в банковских документах

## Описание проекта

Проект, разработанный командой **T1-NERV**, представляет собой микросервисную систему для распознавания текста в банковских документах. Система позволяет загружать документы, извлекать текст с помощью внешнего FastAPI-сервиса и хранить файлы и результаты обработки. Проект построен на современном стеке: Docker, Kafka, MongoDB, MinIO и Spring Boot. Фронтенд находится в отдельном репозитории.

**Основные функции**:
- Загрузка документов с сохранением в MinIO и метаданных в MongoDB.
- Асинхронная обработка документов через Kafka.
- Извлечение текста с помощью FastAPI.
- Хранение результатов в MongoDB.

## Архитектура

Проект состоит из следующих модулей, каждый из которых запускается в Docker-контейнере:

- **MongoDB**: Хранит метаданные документов и результаты обработки.
- **MinIO**: Объектное хранилище для документов.
- **Kafka и ZooKeeper**: Обеспечивают асинхронную передачу сообщений между сервисами.
- **Loader Service**: Spring Boot сервис для загрузки документов, интеграции с MinIO, MongoDB и Kafka.
- **Processor Service**: Spring Boot сервис для обработки документов, взаимодействия с FastAPI и сохранения результатов.
- **FastAPI Service**: Локальный сервис (вне Docker) для извлечения текста.
- **Docker Network**: Общая сеть для взаимодействия сервисов.

## Требования

- Docker и Docker Compose.
- Python (для FastAPI).
- ОС Windows (для `host.docker.internal`).

## Установка и запуск

1. **Клонируйте репозиторий**:
   ```bash
   git clone <repository-url>
   cd <project-directory>
   ```
   
2. **Настройте FastAPI (локально):**
 ```bash
cd <fastapi-directory>
uvicorn main:app --host 0.0.0.0 --port 8000
```


3. **Запустите сервисы:**

**Из корневой папки проекта выполните:**
```bash
docker-compose up --build -d
```


Это развернёт MongoDB, MinIO, Kafka, ZooKeeper, Loader Service (:8080), Processor Service (:8081).
